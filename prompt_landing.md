# Prompt for Fighter Game Landing Page / Full Website

**Role:** You are an expert UX/UI Designer and Frontend Developer. 

**Task:** Design and develop a high-quality, modern, responsive landing page (full website) for the "Fighter Game", a 2D Browser-Based Fighting Game. The site must serve as an end-to-end promotional presentation, showcasing the game's features, characters, mechanics, and providing a direct call-to-action (CTA) to play the game in the browser.

**Aesthetic & Vibe:** 
- **Style:** Clean, modern, highly interactive, and energetic. Think "Modern Arcade" or "Esports Presentation."
- **Color Palette:** Dark theme (#0d0d0d or deep navy/purples) with vibrant neon accents (electric blue, vibrant yellow/gold #fbc531, cyber punk pinks) to highlight action elements.
- **Typography:** A mix of clean modern sans-serif for body text (e.g., Inter, Roboto) and a stylized gaming font (e.g., 'Press Start 2P' or a stylized italic font) for headings.
- **Effects:** Glassmorphism for UI cards, subtle glowing drop shadows, smooth scroll animations, and dynamic hover effects.

---

## 📱 Responsiveness (Crucial)
The website MUST be 100% responsive across all devices:
- **Mobile:** Mobile-first approach. Accordion menus, stacked cards, touch-friendly CTA buttons.
- **Tablet:** Adjusted grid layouts, utilizing side-by-side elements where appropriate.
- **Desktop:** Full-width cinematic hero sections, complex grid features, immersive mouse-hover interactions.

---

## 🗺️ Website Structure & Sections

### 1. Header / Navigation
- Fixed top navigation bar with a glassmorphism effect.
- **Logo:** "Fighter Game" styled text or logo.
- **Links:** Features, Roster, Controls, Multiplayer.
- **CTA:** A glowing "PLAY NOW" button that remains visible.

### 2. Hero Section (Cinematic Entrance)
- **Background:** High-quality game background art or a subtle, looping atmospheric animation (dust particles, neon grid).
- **Headline:** Bold, hook-driven headline (e.g., "Step Into the Arena. Master the Blade.")
- **Subheadline:** "Experience intense 2D browser-based combat. Singleplayer campaigns, WebRTC Multiplayer, and devastating combos."
- **Action:** Primary "PLAY NOW" button and a secondary "Watch Trailer/Gameplay" button.

### 3. Key Features Section
Display features in a sleek 2x2 or 3x1 grid with icons and hover effects.
- **Offline & Online Arena:** Seamlessly transition from fighting intelligent AI to challenging friends via WebRTC Multiplayer.
- **Weapon Arsenal:** Equip unique weapons like the Light Blade, Heavy Blade, Dragon Slayer, and Volt Blade before combat.
- **Devastating Skills:** Unleash ultimate hero abilities like the "Fury Storm" to dominate the battlefield.
- **Cross-Platform:** Full keyboard support for desktop and intuitive on-screen touch D-Pad and action buttons for mobile gamers.

### 4. Character Roster / Hero Selection (Interactive Showcase)
- A carousel or grid showcasing the playable characters:
  - **Default Fighter**
  - **DD**
  - **Haratu**
  - **Luffy**
- Include stats or a brief lore snippet for each character.
- Provide a button to preview their unique fighting stance or special move.

### 5. Weapon Selection
- A visually striking section highlighting the sword selection system.
- Images or icons of the swords beautifully presented with glowing outlines.

### 6. Controls & Mechanics (The 'How to Play')
- A visual layout showing controls for both Desktop and Mobile.
- **Desktop:** Visual representation of keyboard inputs (W A S D for movement, Space for Attack, K for Sword, S for Shield, Q for Skill).
- **Mobile:** A mockup of the mobile UI with the D-Pad and Action Buttons.

### 7. Footer
- Simple and clean.
- Social links, Copyright info, links to "Terms of Service" and "Privacy Policy."
- A final, bottom "ENTER THE ARENA" CTA.

---

## 🛠️ Technical Requirements for the Generated Output:
- Use **HTML5** and **Tailwind CSS** (or pure modern CSS with variables and Flexbox/Grid).
- Provide the complete, single-file HTML structure (or organized folder structure if preferred).
- Write clean, semantic, and accessible markup.
- Implement basic JavaScript for smooth scrolling, mobile menu toggle, and simple animations (e.g., Intersection Observer for fade-in elements).
- Use placeholder images via services like Unsplash or clear image paths where game assets (`/assets/characters/`, `/assets/sword/`) should go.

**End Goal:** Bring this vision to life! The final output should wow the user and feel like a premium, AAA indie game presentation page.
